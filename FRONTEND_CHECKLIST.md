## 🖥️ Ручная проверка (инструкция для пользователя)

### Подготовка

1. **Заполните URL Web App в app.js:**
   ```javascript
   const WEB_APP_URL = "https://script.google.com/macros/d/YOUR_SCRIPT_ID/userweb";
   ```

2. **Откройте index.html в браузере:**
   - Двойной клик на файл или
   - Через VS Code → Open with Live Server

### Проверка 1: Внешний вид

**Что видеть:**
- [x] Шапка с заголовком «💰 Казначейство Либерти»
- [x] Две кнопки: «+ Добавить» и «🔄 Обновить»
- [x] 4 колонки с заголовками и счётчиками (пока 0)
- [x] Пустые колонки готовы к заполнению

**Что проверить:**
```
┌─────────────────────────────────────────────────────────────┐
│ 💰 Казначейство Либерти    [+ Добавить] [🔄 Обновить]      │
├─────────────────────────────────────────────────────────────┤
│ На оплату   │ Согласовано │ Размещено  │ Оплачено           │
│    (0)      │     (0)     │   (0)      │      (0)           │
│ Пусто       │ Пусто       │ Пусто      │ Пусто              │
└─────────────────────────────────────────────────────────────┘
```

### Проверка 2: Интерактивность

#### Кнопка «+ Добавить»
1. Нажмите на кнопку
2. **Ожидается:** открыть модальное окно "Добавить новый платёж"
3. **Проверьте:**
   - Присутствуют поля: Сумма, Контрагент, Приоритет, Срок, Статья бюджета, Назначение, Ссылка
   - Кнопки: "Сохранить" и "Отмена"
   - Нажатие "Отмена" закрывает окно
   - Нажатие Escape закрывает окно

#### Кнопка «🔄 Обновить»
1. Нажмите на кнопку
2. **Ожидается:** 
   - Появится спиннер загрузки
   - Спиннер должен исчезнуть через несколько секунд
   - Если нет ошибок в консоли — Web App URL установлен правильно

### Проверка 3: Цвета и стили

#### Карточки (когда появятся данные)
- Белый фон с левой границей 4px (#2196F3)
- Тень при наведении
- Бейджи приоритетов с нужными цветами

#### Статусные стили (когда будут данные)
Создайте платёж и измените его статус через Web App:
- **status: rejected** → #F5F5F5 фон, #9E9E9E граница
- **Дедлайн в прошлом** → #FFEBEE фон, #EF5350 граница
- **Дедлайн в ближайшие 3 дня** → #FFFDE7 фон, #F9A825 граница

### Проверка 4: Скролл колонок

1. Добавьте много платежей (используя Web App API)
2. Убедитесь:
   - Высота колонок фиксирована
   - Скролл независимый для каждой колонки
   - Шапка остаётся на месте

## 🔧 Настройка перед использованием

### Обязательно!

Отредактируйте `app.js`, строка 4:
```javascript
const WEB_APP_URL = "https://script.google.com/macros/d/YOUR_SCRIPT_ID/userweb";
```

Замените `YOUR_SCRIPT_ID` на реальный ID вашего Web App.

### Опционально

- Измените стили в `style.css` под ваш бренд
- Добавьте иконку сайта в `index.html`:
  ```html
  <link rel="icon" href="favicon.ico">
  ```

## 🐛 Возможные проблемы и решения

### Проблема: Кнопка "Обновить" не загружает данные
**Решение:** Проверьте:
1. Установлен ли `WEB_APP_URL` в app.js
2. Открыт ли Web App на доступ "Anyone"
3. Консоль браузера (F12) → вкладка Console на наличие ошибок CORS

### Проблема: Модальное окно не открывается
**Решение:**
- Проверьте консоль браузера (F12)
- Убедитесь, что все ID элементов в HTML совпадают с селекторами в app.js

### Проблема: Спиннер не исчезает
**Решение:**
- Проверьте сетевую вкладку (F12 → Network)
- Убедитесь, что Web App возвращает JSON
