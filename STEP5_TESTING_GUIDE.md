# Инструкция по тестированию Шага 5

## 🎯 Цель

Убедиться, что кнопки согласования (✓) и отклонения (✗) работают корректно и платежи правильно переходят между статусами.

---

## 📋 Подготовка

1. **Скопируйте Web App URL** из Google Apps Script проекта
2. **Вставьте его** в `app.js` на строку ~10:
   ```javascript
   const SCRIPT_URL = "https://script.google.com/macros/d/YOUR_ID/userweb";
   ```
3. **Убедитесь**, что Google Sheets содержит:
   - Лист "Payments" с колонками: id, amount, counterparty, status, approved, rejection_reason, и т.д.
   - Лист "History" с колонками: payment_id, action_type, old_value, new_value, timestamp

## 🧪 Тестовые сценарии

### Сценарий 1: Видимость кнопок

**Что проверяем:** Кнопки ✓ и ✗ видны только для draft платежей

**Инструкция:**
1. Откройте приложение в браузере
2. Посмотрите на карточки в колонке "На оплату" (первая колонка)
3. На каждой карточке должны быть кнопки: ✓, ✗, ✏️, 🗑️

**При успехе:**
- ✓ На карточках draft видны зелёная (✓) и красная (✗) кнопки
- ✓ На карточках в других колонках эти кнопки НЕ видны

---

### Сценарий 2: Согласование платежа

**Что проверяем:** Платёж переходит в статус "approved"

**Инструкция:**
1. На карточке в primeiro колонке нажмите кнопку ✓ (зелёная галочка)
2. В диалоге подтвердите: "Вы уверены, что хотите согласовать этот платёж?"
3. Нажмите "ОК"

**При успехе:**
- ✓ Появляется сообщение "Платёж согласован"
- ✓ Карточка исчезает из первой колонки
- ✓ Карточка появляется во второй колонке "Согласовано"
- ✓ На карточке уже умер текст нет кнопок ✓ и ✗

**Проверка в Google Sheets:**
1. Откройте Google Sheets → лист "Payments"
2. Найдите согласованный платёж
3. Убедитесь:
   - `status` = "approved"
   - `approved` = TRUE

---

### Сценарий 3: Отклонение платежа (с причиной)

**Что проверяем:** Платёж переходит в статус "rejected" с указанной причиной

**Инструкция:**
1. На карточке в первой колонке нажмите кнопку ✗ (красный крестик)
2. Должно открыться поп-ап "Причина отклонения"
3. Введите причину, например: "Ошибка в контрагенте"
4. Нажмите кнопку "Отклонить"

**При успехе:**
- ✓ Появляется сообщение "Платёж отклонен"
- ✓ Карточка остаётся в первой колонке
- ✓ Карточка становится серой (цвет фона #F5F5F5, граница серая)
- ✓ На карточке больше нет кнопок ✓ и ✗

**Проверка в Google Sheets:**
1. Откройте Google Sheets → лист "Payments"
2. Найдите отклоненный платёж
3. Убедитесь:
   - `status` = "rejected"
   - `approved` = FALSE
   - `rejection_reason` = ваша причина ("Ошибка в контрагенте")

---

### Сценарий 4: Отклонение платежа (без причины)

**Что проверяем:** Кнопка отклонения заблокирована, пока не указана причина

**Инструкция:**
1. На карточке нажмите кнопку ✗
2. Откроется поп-ап "Причина отклонения"
3. НЕ вводите причину, оставьте поле пусто
4. Попытайтесь нажать кнопку "Отклонить"

**При успехе:**
- ✓ Кнопка "Отклонить" неактивна (disabled, т.е. светлая/урезанная)
- ✓ Клик на кнопку не срабатывает
- ✓ Поп-ап остаётся открытым

**Затем:**
1. Введите любой текст в поле
2. Нажмите кнопку "Отклонить"

**При успехе:**
- ✓ Кнопка "Отклонить" теперь активна (яркая)
- ✓ Клик срабатывает
- ✓ Платёж отклоняется с вашей причиной

---

### Сценарий 5: Закрытие поп-апа без отклонения

**Что проверяем:** Платёж не изменяется, если закрыть поп-ап без подтверждения

**Инструкция:**
1. На карточке нажмите кнопку ✗
2. Откроется поп-ап "Причина отклонения"
3. Введите текст: "Причина"
4. Нажмите кнопку "Отмена" или крестик [×]

**При успехе:**
- ✓ Поп-ап закрывается
- ✓ Платёж остаётся в первой колонке со статусом "draft"
- ✓ Платёж НЕ отклоняется
- ✓ В Google Sheets поле `status` остаётся = "draft"

## 🔍 DevTools проверка

### Проверка локального состояния

1. Откройте DevTools (F12)
2. Перейдите на вкладку "Console"
3. Выполните:
   ```javascript
   console.log(state.payments)
   ```
4. Должны увидеть массив платежей с полями `status`, `approved`, `rejection_reason`

### Проверка Network запросов

1. Откройте вкладку "Network"
2. Обновите страницу (F5)
3. Нажмите кнопку согласования или отклонения
4. В Network должны появиться POST запросы:
   - `POST` к SCRIPT_URL с `action=update` — обновление платежа
   - `POST` к SCRIPT_URL с `action=addHistory` — запись истории

### Проверка истории в консоли

```javascript
// Все запросы истории
state.payments.forEach(p => {
  console.log(`ID: ${p.id}, Status: ${p.status}, Approved: ${p.approved}, Rejection Reason: ${p.rejection_reason}`);
});
```

## 🚨 Частые проблемы

### Проблема: Кнопки ✓ и ✗ не видны

**Решение:**
- Проверьте, на какой колонке вы смотрите (должна быть первая "На оплату")
- Проверьте статус платежа в console: `state.payments[0].status`
- Если статус не "draft", кнопки не будут видны

### Проблема: Клик на ✓ ничего не делает

**Решение:**
- Откройте DevTools Console (F12)
- Проверьте, нет ли ошибок (красные сообщения)
- Убедитесь, что SCRIPT_URL заполнена корректно (не пусто, с https://...)
- Проверьте, что в Google Apps Script развёрнут Web App с доступом "Anyone"

### Проблема: После согласования платёж не переместился

**Решение:**
- Проверьте в Google Sheets, что статус изменился на "approved"
- Может быть задержка в API — попробуйте нажать "Обновить"
- Проверьте Network запрос (выполнена ли операция 200 OK)

### Проблема: Поп-ап отклонения не открывается

**Решение:**
- Проверьте наличие элемента: откройте DevTools → выполните `document.getElementById("rejectModal")`
- Должен вернуться элемент (не null)
- Если null — значит HTML не содержит поп-ап (нужно проверить index.html)

### Проблема: Кнопка "Отклонить" в поп-апе всегда активна

**Решение:**
- Проверьте функцию `updateRejectSubmitBtn()` в app.js
- Должна быть строка: `rejectSubmitBtn.disabled = !rejectionReasonField.value.trim();`
- Убедитесь, что обработчик `addEventListener("input", updateRejectSubmitBtn)` присоединён к textarea
