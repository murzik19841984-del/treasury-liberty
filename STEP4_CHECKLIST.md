## ✅ Самопроверка для агента

### Логика создания платежа
```javascript
// При создании (openAddModal):
paymentForm.dataset.paymentId = "";                // нет ID
deleteBtn.style.display = "none";                  // скрыта кнопка удаления
submitBtn.textContent = "Создать";                 // правильный текст

// При успешном создании (handleFormSubmit):
const newPayment = {
  id: result.data.id,
  ...formData,
  status: "draft",                                 // ✓ правильный статус
  approved: false,
  created_at: new Date().toISOString(),
  updated_at: new Date().toISOString()
};
state.payments.push(newPayment);                   // ✓ локальное добавление
renderBoard();                                      // ✓ перерисовываем
// НЕ вызываем fetchAllPayments()
```

### Логика редактирования / обновления
```javascript
// При открытии (openEditModal):
paymentForm.dataset.paymentId = payment.id;        // есть ID
deleteBtn.style.display = "block";                 // видна кнопка удаления
submitBtn.textContent = "Обновить";                // правильный текст

// При успешном обновлении (updatePaymentLocally):
const payment = state.payments.find(p => p.id === paymentId);
Object.assign(payment, newData);                   // ✓ локальное обновление
payment.updated_at = new Date().toISOString();
renderBoard();                                      // ✓ перерисовываем
```

### Валидация
```javascript
validatePaymentForm() {
  const amount = parseFloat(document.getElementById("amount").value);
  const counterparty = document.getElementById("counterparty").value.trim();

  if (!counterparty) {
    showNotification("Поле 'Контрагент' обязательно", "error");
    return false;  // ✓ блокируем отправку
  }

  if (!amount || amount <= 0) {
    showNotification("Сумма должна быть больше 0", "error");
    return false;  // ✓ блокируем отправку
  }

  return true;
}
```

### Блокировка кнопок
```javascript
// В начале handleFormSubmit:
submitBtn.disabled = true;   // ✓ блокируем
deleteBtn.disabled = true;   // ✓ блокируем

// В конце (finally блок):
submitBtn.disabled = false;  // ✓ разблокируем
deleteBtn.disabled = false;  // ✓ разблокируем
```

## ⚠️ Возможные проблемы

### Проблема 1: Кнопка "Удалить" всегда видна или всегда скрыта
**Решение:**
- Проверьте код в `openAddModal()` и `openEditModal()`
- Убедитесь, что `deleteBtn.style.display = "none"` в openAddModal
- Проверьте, что `deleteBtn.style.display = "block"` в openEditModal

### Проблема 2: После создания платежа данные не появляются
**Решение:**
- Проверьте, что результат API имеет поле `success: true`
- В консоли (F12) посмотрите ответ API
- Убедитесь, что вызывается `renderBoard()` после создания

### Проблема 3: Кнопки не блокируются
**Решение:**
- Проверьте, что в handleFormSubmit установлены `submitBtn.disabled = true`
- Убедитесь, что в блоке finally разблокируются: `submitBtn.disabled = false`
- Проверьте CSS стили для `:disabled`

### Проблема 4: Валидация не работает
**Решение:**
- В консоли (F12) проверьте, вызывается ли `validatePaymentForm()`
- Убедитесь, что функция возвращает `true` или `false`
- Проверьте селекторы полей (id="amount", id="counterparty" и т.д.)
