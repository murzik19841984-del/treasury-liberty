## ⚙️ Технические детали

### State структура
```javascript
state = {
  payments: Array,              // все платежи
  selectedPayments: Set,        // выбранные ID (быстрый поиск)
  rejectingPaymentId: String    // для поп-апа отклонения
}
```

### API запросы
```
POST /batchUpdate
  - ids: Array<string>
  - status: "approved" | "placed" | "paid" | "rejected" | "deleted"

POST /addHistory (несколько запросов, по одному на платеж)
  - payment_id: string
  - action_type: "batch_update"
  - old_value: string (старый статус)
  - new_value: string (новый статус)
```

### Performance
- ✅ Set для O(1) поиска в selectedPayments
- ✅ Локальное обновление без full refetch
- ✅ История добавляется асинхронно (не блокирует UI)
- ✅ Batch операция через один API запрос
