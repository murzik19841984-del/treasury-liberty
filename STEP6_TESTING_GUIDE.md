# Инструкция по тестированию Шага 6 - Групповые операции

## 🎯 Цель

Убедиться, что чекбоксы, "Выбрать все" и плавающая панель работают корректно, и можно выполнять массовые операции со статусами платежей.

---

## 📋 Подготовка

Убедитесь, что:
1. SCRIPT_URL заполнена в app.js
2. Есть минимум 3-5 платежей со статусом "draft"
3. Есть платежи в других колонках

---

## 🧪 Тестовые сценарии

### Сценарий 1: Видимость плавающей панели

**Что проверяем:** Панель появляется при выборе платежей и исчезает при отмене

**Инструкция:**
1. Откройте приложение
2. Нажмите чекбокс на первой карточке в колонке "На оплату"
3. Проверьте внизу экрана

**При успехе:**
- ✓ Плавающая панель появляется с анимацией
- ✓ На панели показано "1 выбран"
- ✓ На панели показана сумма платежа (например, "Сумма: 50,000 ₽")
- ✓ Чекбокс на карточке отмечен

**Затем:**
1. Нажмите кнопку "Отмена" на панели

**При успехе:**
- ✓ Панель исчезает
- ✓ Чекбокс снимается
- ✓ Выделение очищается

---

### Сценарий 2: Расчет суммы при выборе нескольких платежей

**Что проверяем:** Сумма корректно складывается

**Инструкция:**
1. Выберите три платежа с суммами: 100 000, 50 000, 25 000 ₽
2. Посмотрите на pannель внизу

**При успехе:**
- ✓ На панели показано "3 выбрано"
- ✓ На панели показано "Сумма: 175,000 ₽" (правильная сумма)
- ✓ При отмене одного платежа сумма пересчитывается

---

### Сценарий 3: Правильное склонение чисел

**Что проверяем:** Русское склонение ("выбран", "выбрано", "выбрана")

**Инструкция:**
1. Выберите 1 платеж
2. Посмотрите текст на панели

**При успехе:**
- ✓ "1 выбран" (не "выбрано", не "выбрана")

**Затем:**
1. Выберите 2 платежа

**При успехе:**
- ✓ "2 выбраны" (не "выбран", не "выбрано")

**Затем:**
1. Выберите 5 платежей

**При успехе:**
- ✓ "5 выбрано" (правильное окончание)

---

### Сценарий 4: Кнопка "Выбрать все" в колонке

**Что проверяем:** Кнопка ☑️ в заголовке колонки выбирает все платежи в этой колонке

**Инструкция:**
1. Откройте приложение
2. Посмотрите заголовок колонки "На оплату"
   - Должна быть иконка ☑️ справа от названия
3. Нажмите на иконку ☑️

**При успехе:**
- ✓ Все видимые платежи в этой колонке получают галочки
- ✓ На панели показано количество (например, "5 выбрано")
- ✓ На панели показана общая сумма всех платежей в колонке

**Затем:**
1. Нажмите на другую колону (например, "Согласовано")
2. Нажмите ☑️ в её заголовке

**При успехе:**
- ✓ Платежи из первой колонки остаются выбранными
- ✓ Добавляются платежи из второй колонки
- ✓ Количество и сумма обновляются (суммируются платежи из обеих колонок)

**Затем:**
1. Нажмите "Отмена"

**При успехе:**
- ✓ Панель закрывается
- ✓ Все чекбоксы снимаются
- ✓ В Google Sheets ничего не изменилось

---

### Сценарий 5: Массовое согласование (основной тест!)

**Что проверяем:** Три платежа одновременно переместились в колонку "Согласовано"

**Инструкция:**
1. Выберите 3 платежа в колонке "На оплату"
2. Посмотрите на информацию панели (количество и сумма)
3. Нажмите кнопку "✓ Согласовать" на панели

**При успехе:**
- ✓ Платежи исчезают из колонки "На оплату"
- ✓ Платежи появляются в колонке "Согласовано"
- ✓ Панель исчезает
- ✓ Появляется сообщение "Обновлено 3 платежей"
- ✓ Счетчик "На оплату" уменьшился на 3
- ✓ Счетчик "Согласовано" увеличился на 3

**Проверка в Google Sheets:**
1. Откройте Google Sheets → лист "Payments"
2. Найдите три платежа, которые согласовали
3. Убедитесь:
   - ✓ `status` = "approved"
   - ✓ `updated_at` = текущая дата/время

**Проверка истории:**
1. Откройте лист "History"
2. Посмотрите внизу таблицы
3. Должно быть 3 новых строки:
   ```
   | payment_id | action_type  | old_value | new_value |
   |------------|--------------|-----------|-----------|
   | id1        | batch_update | draft     | approved  |
   | id2        | batch_update | draft     | approved  |
   | id3        | batch_update | draft     | approved  |
   ```

## 🔍 DevTools проверка

console.log(state.selectedPayments)

// 3

### Проверка суммы вручную
```javascript
let sum = 0;
Array.from(state.selectedPayments).forEach(id => {
  const payment = state.payments.find(p => p.id === id);
  sum += payment.amount;
});
console.log(sum)  // должна совпадать с панелью
```

### Network запросы
1. Откройте вкладку Network
2. Выберите платежи и нажмите кнопку статуса
3. Должны появиться POST запросы:
   - POST к SCRIPT_URL с `action=batchUpdate`
   - Несколько POST к SCRIPT_URL с `action=addHistory` (по одному на каждый платеж)

## 🚨 Частые проблемы

### Проблема: Сумма считается неправильно

**Решение:**
- Проверьте, что все платежи имеют поле `amount` (число, не строка)
- Убедитесь, что `formatMoney()` корректно форматирует числа

```javascript
// Проверить в console:
state.selectedPayments.forEach(id => {
  const p = state.payments.find(x => x.id === id);
  console.log(p.amount, typeof p.amount);  // должен быть number
});
```

### Проблема: Кнопка "Выбрать все" не работает

**Решение:**
- Проверьте, что кнопки существуют: `document.querySelectorAll(".column-select-all")`
- Проверьте, что событие сработало: добавьте `console.log()` в обработчик
- Убедитесь, что в колонке есть карточки

### Проблема: История не записывается

**Решение:**
- В console проверьте наличие ошибок при вызове `addHistory()`
- Убедитесь, что SCRIPT_URL не пусто
- Проверьте в Google Sheets, что лист "History" существует

### Проблема: Платежи не переместились, но история записалась

**Решение:**
- Нажмите кнопку "Обновить"
- Или обновите страницу (F5)
- Проверьте в Google Sheets, что статус действительно изменился (может быть задержка синхронизации)
